# 001-量子科技研究人员数据接入

## 工作概述

本次工作将量子科技领域的研究人员数据（497条）接入前端系统，实现了完整的人才库功能模块，包括数据转换、筛选、搜索、分页展示等纯前端实现。

## 实现功能

### 1. 数据转换层
- **Python 转换脚本**: `scripts/convert_parquet.py`
  - 读取 5 个 Parquet 文件
  - 数据清洗（职称标准化、机构标准化、研究标签提取）
  - 生成 `public/data/people.json` (529KB)

### 2. 前端数据层
- **类型定义**: `src/types/people.ts`
  - 定义 Researcher、TitleLevel、Institution 等核心类型
  - 职称配置映射 (TITLE_CONFIG)
  - 机构配置映射 (INSTITUTION_CONFIG)

- **数据适配器**: `src/adapters/researcherAdapter.ts`
  - 原始 Parquet 数据 → 标准化 Researcher 对象
  - 职称标准化（pi/professor/associate/postdoc/phd/other）
  - 研究标签提取（基于关键词匹配）
  - 教育背景/工作经历解析（从 biography 提取）

- **API 封装**: `src/api/researchers.ts`
  - 支持本地 JSON 数据加载
  - 支持后端 API 切换（预留）
  - 统计信息查询
  - 全局搜索功能

- **React Hook**: `src/hooks/useResearchers.ts`
  - useResearchers: 列表数据获取、筛选、分页
  - useResearcherDetail: 单个人员详情
  - useResearcherStatistics: 统计数据
  - useResearcherSearch: 带防抖的搜索

### 3. UI 组件层

| 组件 | 路径 | 功能 |
|------|------|------|
| InstitutionBadge | `components/researcher/InstitutionBadge.tsx` | 机构徽章（带颜色和图标） |
| TitleBadge | `components/researcher/TitleBadge.tsx` | 职称徽章 |
| ResearchTagCloud | `components/researcher/ResearchTagCloud.tsx` | 研究方向标签云 |
| ResearcherCard | `components/researcher/ResearcherCard.tsx` | 人员卡片（支持拖拽） |
| ResearcherDetailModal | `components/researcher/ResearcherDetailModal.tsx` | 详情弹窗 |

### 4. 页面层
- **人才库页面**: `src/pages/Researchers.tsx`
  - 响应式网格布局（1/2/3列）
  - 机构筛选（可多选）
  - 职称筛选（可多选）
  - 邮箱/简介存在性筛选
  - 实时搜索（姓名、机构、研究方向）
  - 分页加载（每页30条）
  - 活跃筛选标签展示

## 技术特点

### 纯前端实现
- 无需后端 API，所有数据在浏览器端处理
- 筛选、搜索、分页均在内存中完成
- 响应速度极快（497条数据量小）

### 数据流设计
```
Parquet → convert_parquet.py → people.json
                                    ↓
                           fetch('/data/people.json')
                                    ↓
                           adaptRawToResearcher()
                                    ↓
                           useResearchers() Hook
                                    ↓
                           Researchers.tsx 页面
```

### 筛选逻辑
- 机构筛选：多选 OR 逻辑
- 职称筛选：多选 OR 逻辑
- 搜索查询：姓名、英文名、机构、部门、职称、研究方向、标签
- 存在性筛选：有邮箱（非空）、有简介（长度≥10）

### 分页设计
- 每页 30 条
- 加载更多按钮
- 筛选条件变化时重置到第1页
- 避免重复数据（基于 ID 去重）

## 遇到的问题与修复

### 问题1: 筛选无响应
**原因**: `useResearchers` hook 只读取 `initialFilters` 一次，不响应后续变化
**修复**: 添加 `useEffect` 监听 `initialFilters` 变化

### 问题2: 邮箱/简介筛选无效
**原因**: 空字符串 `""` 不等于 falsy，筛选条件判断不准确
**修复**: 改为检查 `email.trim() !== ''` 和 `biography.trim().length >= 10`

### 问题3: 分页数据重复
**原因**: 切换筛选后页码未重置，导致数据叠加
**修复**: 筛选条件变化时自动 `setPage(1)`
