# 报告结构详细指南

投研报告必须遵循10章节结构。每章都有一个核心观点——读者读完该章后应能用一句话说出"这节告诉我什么"。

---

## 设计原则

报告以**读者决策**为中心，不以数据来源为中心。paper-analysis/people-intel/market-intel 的输出是原材料，最终报告要综合进以下结构，而非简单拼接三段。

**格式适应性**：数据充分时使用结构化表格；数据不足时如实注明"该维度数据不足，无法评估"，不用模糊内容填充格式框架。所有原始数字（h-index、融资金额、实测指标）完整保留，不得摘要性删减。

---

## 各章节详细规范

### 1. 执行摘要

独立可读的决策文档（不是前言）：3-5个最重要的结论、支撑每条结论的关键数字（每条附来源）、对读者决策最直接的含义。

判断标准：只看这一章，读者能做出80%的判断。

### 2. 行业定义与边界

口径声明：本报告的范围边界（包括什么/不包括什么）、整体数据截止日期、主要数据来源。若同一指标存在多种统计口径（如市场规模），在此统一说明差异，后文不再重复解释。

### 3. 现状层

三个维度横向打通，不要三段各自独立：

- **技术成熟度**（来自 paper-analysis）：各路线实测指标 + TRL推断范围 + 推断依据 + 关键突破（每条附论文来源）+ 主要瓶颈
- **商业化进展**（来自 market-intel）：谁在赚钱、赚谁的钱、商业模式是否跑通、关键融资事件（附来源日期）
- **人才与生态**（来自 people-intel）：核心机构与代表人物（h-index来自工具，不编造）、产业化流向、供应链关键节点

**横向打通（必填）**：用1-2段话明确指出——哪里技术成熟但商业未跑通；哪里资本已聚集但人才空白；哪里政策支持但生态缺失。这是三路数据综合后才能得出的洞察，纯粹区别于三段分别汇报的价值所在。

### 4. 动态层（驱动力与阻力）

驱动力和阻力篇幅大致对等——只写驱动力的报告是残缺的，只有正面信号的报告读者不会信任。各自分三个时间层：

| 层级 | 驱动力（附信号来源） | 阻力（注明类型） |
|------|------|------|
| 短期（1-2年内会触发的具体事件） | ... | 技术瓶颈/商业模式问题/监管障碍 |
| 中期（3-5年结构性趋势） | ... | ... |
| 长期不确定性（可能改变行业逻辑的变量） | ... | ... |

阻力类型不同，解法完全不同——必须区分标注（技术瓶颈和监管障碍的应对策略完全不一样）。

### 5. 竞争格局（权力结构）

不是公司名单，而是权力结构分析：价值在哪里沉淀、谁有定价权、壁垒来源是什么（技术/数据/生态/监管）、格局是否稳定。落脚点：如果我是这个行业的玩家，我最担心什么。

### 6. 前沿信号

市场尚未充分定价的信息增量（这部分体现研究深度，是报告的差异化价值）：

- **学术前沿突破**（来自 paper-analysis，附论文来源）
- **人才流向异动**（来自 people-intel：谁离开了学术界/谁在筹建新机构；**每条事件附时间节点 YYYY-MM，禁止只写方向不写时间**）
- **早期融资方向**（天使/种子/Pre-A 轮，预示下一波热点；附融资时间）
- **监管态度微变**
- **新兴玩家雷达（Deal Sourcing 专项）**：

  | 信号类型 | 发现方式 | 判断价值 |
  |---------|---------|---------|
  | 近 2 年内首次融资的初创公司 | news-market 融资扫描 | 最早期信号，估值尚低 |
  | 高校/科研院所孵化团队 | people-intel + news-market 交叉 | 技术真实性高 |
  | 大厂量子团队离职创业 | people-intel 人才流动 + Web 时间核查 | 工程化执行力强 |
  | 海归核心科学家创业 | people-intel DB + Web 动态补充 | 国际资源与人脉 |
  | 已有专利/论文产出但尚未融资 | paper-analysis + Web 搜索 | 隐形早期标的 |

  > **注意**：对头部已上市/已知公司的动态分析对 Deal Sourcing 价值有限——真正的 Alpha 在上述早期信号里。本章必须包含至少一个新兴玩家信号，或明确说明"本次扫描未通过工具发现符合条件的早期标的"。

### 7. 情景分析

单点预测（"2030年市场规模将达到XXX亿"）无法表达不确定性，对读者误导大于帮助。改用情景回答"如果X发生，会怎样"：

每个情景包含：**名称** + **核心假设**（2-3个）+ **对应的技术/商业/政策表现** + **判断正在走向哪个情景的早期可观测指标**。

至少包含一个悲观/不及预期情景——没有悲观情景的分析是不完整的。

### 8. 风险矩阵

风险不是免责声明，是分析的一部分。每个风险包含：

- **影响程度**（高/中/低）× **发生概率**（高/中/低）
- **触发条件**：什么事件会让这个风险变成现实
- **早期预警信号**：具体可观测的指标，提示风险正在演变
- **应对建议**：如果风险触发，投资人/从业者该怎么办

### 9. 对读者的具体建议

泛泛的"建议关注XX领域"没有行动指向，读者看完不知道该怎么做。每条建议必须包含：

> **行动**：[做什么，具体到标的/方向]  
> **时间窗口**：[不是"未来"而是具体触发条件，如"当X事件发生时"]  
> **前提条件**：[哪个假设必须成立，这条建议才有效]  
> **退出信号**：[什么情况出现，说明这条建议已失效，需要重新评估]

### 10. 数据说明与局限性

让读者知道哪些结论可直接用、哪些需核实：

- 数据截止时间
- 主要来源及可靠性评估（按置信度分级）
- 关键数据缺口（哪些重要信息没有覆盖）
- 依赖较强假设的结论列表
