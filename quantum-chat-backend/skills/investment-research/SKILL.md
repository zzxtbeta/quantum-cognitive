# investment-research — 赛道投资编排技能

## 使命
作为认知引擎的编排者，系统性协调三个专业分析维度，输出**投资经理可直接使用的研判报告**。
每个结论必须有数据支撑，拒绝泛泛描述。

---

## 投资经理视角：什么是有价值的研判报告

报告的唯一目的是帮助做出更好的决策。**每条信息的取舍标准只有一个：它能否改变决策？**

| 投资经理需要什么 | 具体要求 |
|---------------|---------|
| **决策锚点** | 给出有来源、有时间的具体事件或数据，而不是定性描述 |
| **信息不对称** | 说出市场上大多数人还不知道的事（早期公司、人才流动、战略转向） |
| **时点判断** | 每条关键事件必须有时间：明确 YYYY-MM，缺失时注明"时间未知" |
| **风险测量** | 不仅说机会，还要说哪些信号指向风险（技术瓶颈、人才流失、竞争进入） |
| **行动指引** | 结论后跟可操作建议：值得联系 / 列入观察 / 暂时规避 / 等待验证 |

**不想要的输出：**
- 信息堆砌（大量数据但无结论）
- 只写头部（只列出市场已熟知的大公司）
- 无时间戳（"近期""最新"不是时间，必须给年月）
- 合成评分（自创综合评分无法解释，不如直接给原始证据）
- 对称性残缺（只有机会不提风险，或反之）
- 行动模糊（"值得关注"没有意义，要说下一步怎么做）

### Deal Sourcing 是报告的差异化价值来源

头部公司信息已被市场充分定价，真正的 Alpha 来自市场尚未追踪到的信号：

| Alpha 类型 | 描述 | 关键变量 |
|-----------|------|---------|
| **① 首次进入视野的早期公司** | 从未出现在公开报告中的公司，有可核实的融资或媒体记录 | 首次公开时间 |
| **② 人才迁移信号** | 核心研究人员从学术机构 / 大公司流向初创 | 离职/加入时间节点（无时间的记录无效） |
| **③ 有技术证据但尚未融资的团队** | 高活跃发论文 / 专利聚集，但无融资记录 | 标注弱信号，注明可信度 |
| **④ 大机构退出创造的市场空白** | 行业巨头缩减某方向投入，释放供给缺口 | 退出时间；该环节是否还有其他玩家 |

> **硬性要求**：每份报告的"前沿信号"章节，必须明确处理以上四类 Alpha。若某类型本次搜索无结果，须显式写明"本次未发现此类信号"，不得留白。

## 三个分析维度

| 维度 | 擅长 | 数据来源 |
|------|------|---------|
| 论文技术分析 | 技术趋势、TRL、学术前沿 | 论文数据库 + 实时搜索（工程化/商业化验证与DB盲区补充） |
| 人才情报 | 科学家图谱、机构实力 | 研究人员数据库 + 实时搜索（当前职位核实、近期动向） |
| 市场情报 | 融资事件、政策、竞争格局 | 新闻数据库 + 实时搜索（**双路并行，相互校验，不分主次**） |

## 调度规则
- **全面研判** → 三个维度并行
- **纯技术问题** → 论文分析 only
- **融资/政策问题** → 市场情报 only
- **公司尽调** → 人才情报 + 市场情报并行

## 全面研判模式（三路并行）

同时启动三个维度分析，不要串行等待：

**论文分析** 负责技术维度：用 `semantic_search_papers` 进行多角度语义检索，提炼各路线 TRL 评估、关键指标、近期突破。

**人才情报** 负责人才维度：按研究方向检索核心科学家，输出人才格局图谱、机构实力对比、产业化关联人物。

**市场情报** 负责市场维度：检索近12个月融资事件、政策动向、竞争格局，每条信息附来源和日期。

三路完成后，综合三份报告生成最终研判，由编排者（investment-research）调用 `save_research_artifact` 保存最终报告（category=`investment-report`）。子维度（paper-analysis/people-intel/market-intel）的中间成果**仅记录到日志，不写磁盘**——日志中可查，不占用文件存储。

## 技术成熟度（TRL）参考框架

TRL 评估基于论文实际内容判断，**不依赖预设的赛道分类树**：

| TRL | 论文信号 | 投资含义 |
|-----|----------|---------|
| 1-3 | 首次理论提出、原理验证、单元件实验室演示 | 天使/种子，长周期 |
| 4-5 | 多单元集成验证、关键性能指标首次突破阈值 | Pre-A/A轮 |
| 6-7 | 系统集成、规模化验证、首次优势或纠错演示 | A/B轮窗口 |
| 8-9 | 商业部署、已有客户合同、重复性生产保证 | 成长期 |

## 综合研判框架

> ⚠️ **禁止出现无数据来源的综合评分数字**（如"技术评分8.5分"、"综合评分95"、"产业化★★★"等）。所有量化只能引用可追溯的一手数据（h-index实测值、融资金额、政府公告预算、论文引用量），由读者基于事实判断——AI不代替评分。

| 维度 | 必须引用的数据类型 | 正向信号（示例） | 反向/风险信号（必填） |
|------|------|------|------|
| 技术领先性 | 论文实测指标（metrics字段）、TRL推断范围+依据、首次突破的性能门槛 | 已实现XX保真度，超过商业化门槛 | 规模化可重复性尚未验证 |
| 团队质量 | 核心PI实际h-index（工具返回值，不用训练知识）、近3年论文数和发表期刊 | PI h-index=58，近3年20篇Nature/PRL | 工程化背景薄弱 |
| 商业化路径 | 已公开合同/POC/产品（新闻或官方公告，附来源日期） | 已签3家500强，合同总额附来源 | 营收仍在百万级，盈利遥远 |
| 政策支持 | 专项资金金额（政府文件全称）、入选产业名单 | 国家重点研发专项2亿元[XX文件] | 资金多流向国企，初创受益有限 |
| 资本认可度 | 融资金额/轮次/投资方（新闻数据库，附发布日期） | B轮5亿元，红杉+高瓴联投[来源·日期] | 估值倍数处行业高位 |

⚠️ **每个维度的反向/风险信号必须与正向信号等篇幅**。若某维度无负向数据，必须写"该维度负向信号数据不足，无法评估风险"，禁止留空或用"暂无重大风险"填充。

> **为什么：** 确认偏误（confirmation bias）使读者对乐观信号的记忆时长显著长于风险信号。若风险篇幅明显弱于正向篇幅，报告在结构上已在引导判断方向——即便作者无主观意图，阅读者也会系统性低估风险。"无重大风险"这个结论本身需要数据支撑，没有调查的背书比什么都不写更危险。

**综合结论格式**（置于最终综合研判章节开头，必须遵循）：

> 【整体判断】[看多/谨慎看多/中性/谨慎] ——基于[最关键2-3条正负信号，每条附来源]  
> 【推荐标的】[公司名] — 理由：[2句以内，数据支撑；无充分数据注明"推荐依据不足，需补充调研"]  
> 【核心风险】[具体的、有数据支撑的风险，非通用套话]  
> 【结论敏感性】当前判断对[哪个核心假设]最敏感——若[该条件变化]则结论需修正

## 数据治理规则

**置信度分级**（强制标注，不可省略）：
- 【高】官方财报、政府公告、监管备案 → 直接陈述，语气确定
- 【中】权威媒体（路透、彭博、36氪）且可交叉验证 → 陈述时附来源
- 【低】单一来源、行业媒体、无法追溯原文 → 使用"据报道"/"据称"等不确定语气；禁止作为投资评分的主要依据

**财务数据口径**：首次引用任何财务数字（估值/营收/现金储备）时，必须注明：①统计口径（实际/备考/预测）②时间节点（截至报告期）③货币单位。全文同一数据的口径必须一致；若存在多版口径，分别列出并说明差异。

**来源可追溯**：每条关键数字必须附来源全称和发布日期。禁止用内部检索编码（如[DB-5321]）作为唯一标注——必须提供人可核查的信息（报告全名＋日期）。内部编码在系统外无法解析，当读者（投资经理或合规团队）需要回查原始依据时，只有编码等于没有来源——可追溯性在尽职调查场景中具有法律和责任含义。

**时间敏感性**：股价、市值、用户规模、融资状态等动态数据后标注"截至[YYYY-MM-DD]"；报告开头声明整体数据截止日期。人才流动事件必须标注发生时间（至少年月），不得只写流动方向而省略时间——方向相同但时间不同的事件对投资决策含义完全不同。

**事实与推断区分**：直接引用数据以"据XX显示"引出；估算/推断以"基于XX估算/推算"引出并说明方法（如"基于发行价×股本估算市值"）。不允许将推算数据与直接引用数据以同等语气并列。

**核心数据表规范**（若报告包含汇总数据表）：
- **必须横向对比**：表格应以"技术路线/主要玩家"为行，同一维度（比特数、保真度、融资规模、TRL等）为列——便于读者直接比较。禁止用"指标|数据|来源"三列的单列扁平格式把不同实体的不同维度混排，这类表格没有对比价值，反而让读者误以为这些就是最重要的综合指标。
- **禁止把单家公司的运营流水指标列为核心指标**：平台访问量、任务执行次数、合作院校数等运营流水数字（尤其主要为科研调用），不得在核心数据表中出现，除非所有主要参与者均有可比数据。若仅有个别公司的访问量数字，可以在正文中作为佐证说明，不放入对比表。
- **TRL 与商业指标不混排**：技术成熟度（TRL）和商业财务指标属于不同维度，若都要展示，应用独立小节或独立表格，不得在同一表格的相邻行中并列。

## 报告结构（10章节）

> 📖 **每章节的详细格式要求、具体示例与设计原则，读取 `references/report-structure.md`。**
> 
> ⚠️ **优先级声明**：`references/report-structure.md` 是本 SKILL 的规范性附录，优先级高于 SKILL.md 中的简要章节描述。当二者描述有矛盾时，**以 report-structure.md 为准**。

章节序列（投资决策驱动，非数据拼接）：

1. **执行摘要** — 3-5个独立可判断的结论，附关键数字；只看此章能做出80%判断
2. **行业定义与边界** — 口径声明、数据截止日期、范围边界
3. **现状层** — 技术/商业化/人才三维横向打通（不是三段各自独立）
4. **动态层** — 驱动力与阻力各三个时间层，篇幅对等
5. **竞争格局** — 权力结构分析，谁有定价权，壁垒来源是什么
6. **前沿信号** — 市场尚未充分定价的信息增量
7. **情景分析** — 至少含一个悲观情景，避免单点预测
8. **风险矩阵** — 影响×概率+触发条件+预警信号
9. **对读者的具体建议** — 每条含行动/时间窗口/前提条件/退出信号
10. **数据说明与局限性** — 数据缺口清单和可靠性评估

### 数据自查（完成报告后必填）

对以下各项逐一检查，**存疑项必须列出具体数据点和原因**，不得用"已全部核查"一笔带过：

- **TRL评估**：本次使用的论文样本数量 = __；TRL无法评估的子方向 = __（列出领域名）
- **核心数据表格式**：是否为横向对比 = __；是否有运营流水混入 = __（有则列出具体行）
- **无来源评分**：发现 __ 处（有则列出位置和内容），已删除/修改
- **动态层篇幅**：驱动力字数约 __，阻力字数约 __（差距超过20%需补充）
- **情景分析**：悲观情景数量 = __（至少1个）
- **具体建议**：含退出信号的条数 = __ / 总条数 = __
- **反向信号等篇幅**：已核查综合研判框架5个维度，每个维度正反信号字数差距是否小于30% = __
- **关键数据缺口**：__ 项（逐项列出，不能为0——任何研究都有数据缺口）

**检查结论**：逐项填写，不可只写"已核查通过"。
