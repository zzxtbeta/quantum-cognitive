# 量子赛道深度调研——能力移植手册

> **用途说明**：本文档是一份完整的能力移植包，用于将量子赛道深度调研能力复制给其他 AI 智能体（如 OpenClaw）。  
> 将本文档内容完整提供给目标 AI，引导其沉淀为三个可调用的 Skills：`paper-research`、`people-intel`、`news-market`。

## 零、能力全景速览

本能力包含 **三个并行调研子技能**，以及一套将三路结果融合输出的编排方案。

| 技能 | 核心工具 | 数据源 | 典型耗时 |
|------|----------|--------|---------|
| `paper-research` | 量子论文数据库 REST API | 460 篇顶刊顶会论文（持续更新） | 60–120s |
| `people-intel` | 研究员数据库 REST API | 中国量子赛道核心研究员档案 | 30–60s |
| `news-market` | Web 搜索（Tavily 等） | 实时网络信息 + 市场数据 | 30–90s |

**三技能并行后由编排层融合**，生成 2500 字以上投资级报告。

### 1.2 论文数据库 API — `GET /papers`

**功能**：在 460 篇量子赛道顶刊论文中检索与分析。

**请求参数**：

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `domain_ids` | string | — | 逗号分隔的领域 ID，如 `10,11`。不传则全库搜索 |
| `page` | int | 1 | 页码，从 1 开始 |
| `page_size` | int | 20 | 每页数量，建议 60–80 进行批量扫描 |
| `sort_by` | string | `publish_date` | 排序字段：`publish_date` / `title` |
| `sort_order` | string | `desc` | 排序方向：`desc` / `asc` |
| `query` | string | — | 关键词全文搜索，支持中英文 |
| `author_name` | string | — | 按作者姓名搜索，支持模糊匹配 |
| `time_range` | string | — | 时间过滤，如 `2023-01-01,2025-12-31` |
| `include_stats` | bool | false | 为 `true` 时返回领域统计摘要 |

**调用示例**：

```bash
# 示例1：获取超导量子最新论文（domain_ids 从 /gold/domains 查得）
curl -H "X-API-Key: xK7mP9nQ2wR5tY8uI1oL4aS6dF3gH0jK" \
  "https://www.gravaity.ai/datalake/api/papers?domain_ids=10&page=1&page_size=60&sort_by=publish_date&sort_order=desc&include_stats=true"

# 示例2：关键词搜索量子纠错最新进展
curl -H "X-API-Key: xK7mP9nQ2wR5tY8uI1oL4aS6dF3gH0jK" \
  "https://www.gravaity.ai/datalake/api/papers?query=量子纠错+逻辑量子比特&page_size=30&sort_by=publish_date"

# 示例3：全库批量扫描（每次80篇，翻3页覆盖主流数据）
curl -H "X-API-Key: xK7mP9nQ2wR5tY8uI1oL4aS6dF3gH0jK" \
  "https://www.gravaity.ai/datalake/api/papers?page=1&page_size=80&sort_by=publish_date&sort_order=desc"

# 示例4：按作者姓名搜索论文
curl -H "X-API-Key: xK7mP9nQ2wR5tY8uI1oL4aS6dF3gH0jK" \
  "https://www.gravaity.ai/datalake/api/papers?author_name=潘建伟&page_size=20&sort_by=publish_date"
```

## 二、技能一：paper-research

```skill
---
name: paper-research
description: Use this skill when you need to analyze quantum technology papers, understand research trends, evaluate technology readiness, identify key breakthroughs, or map the technical landscape of quantum computing, communication, or sensing. Activates for any paper-based technical intelligence request.
license: MIT
---
```

### 使命

从量子赛道顶刊顶会论文数据库中提炼**投资级技术洞察**，而非表面摘要。  
目标：让投资经理读完报告后能准确判断——哪个技术路线最有前景、谁在做、有多成熟。

---

### 常见领域 ID 参考表

> 调用前先执行 `GET /gold/domains` 获取最新完整映射。以下为常用 ID，仅供参考：

| 领域方向 | 关键词 | 说明 |
|----------|--------|------|
| 超导量子计算 | qubit, transmon, superconducting | 工程化最成熟，国内论文最多 |
| 离子阱量子计算 | ion trap, trapped ion | 保真度最高，扩展性是难题 |
| 光量子计算 | photonic, photon, boson sampling | 室温运行，通信/计算双路径 |
| 量子纠错 | error correction, surface code, logical qubit | 容错计算的核心 |
| 量子通信/QKD | quantum key distribution, QKD | 中国国际领先方向 |
| 量子传感 | quantum sensing, magnetometer, gravimeter | 近年最快商业化方向 |
| 中性原子 | neutral atom, Rydberg | 2023–2024 年爆发式增长 |
| 硅量子点 | silicon spin qubit, quantum dot | 半导体工艺兼容的长期路线 |

---

### 标准工作流（四步）

#### Step 1：获取领域 ID 树
```
GET /gold/domains?min_paper_count=0
→ 解析响应，建立 name→id 映射表
```

#### Step 2：分领域批量扫描（核心！不要只调一次）
```
# 请求目标：每个主要赛道至少获取60篇，按 publish_date 降序
GET /papers?domain_ids=<超导ID>&page=1&page_size=60&sort_by=publish_date&sort_order=desc
GET /papers?domain_ids=<离子阱ID>&page=1&page_size=60&sort_by=publish_date&sort_order=desc
GET /papers?domain_ids=<光量ID>&page=1&page_size=60&sort_by=publish_date&sort_order=desc
GET /papers?domain_ids=<中性原ID>&page=1&page_size=60&sort_by=publish_date&sort_order=desc
# 如需更多，翻 page=2, page=3
```

#### Step 3：专题深挖（针对投资焦点）
```
# 针对用户最关心的技术点，按关键词搜索
GET /papers?query=量子纠错+逻辑量子比特&page_size=30&sort_by=publish_date
GET /papers?query=量子优势+实验演示&page_size=20&sort_by=publish_date&sort_order=desc
# 按作者搜索
GET /papers?author_name=<科学家姓名>&page_size=20&sort_by=publish_date
```

#### Step 4：汇总输出（按模板）
- 统计每条赛道的论文总数（`total` 字段）
- 提取近期论文作为核心引用
- 从 `metrics` 字段提取可量化数据（量子比特数、保真度、相干时间）
- 从 `key_contributions` 字段识别技术突破

---

### 关键字段提取规则

| 字段 | 提取要点 |
|------|----------|
| `research_problem.summary` | 一句话分类这篇论文解决什么问题 |
| `tech_route.summary` | 判断技术路线归属（超导/离子阱/光量子等） |
| `key_contributions` | 寻找有具体数字的硬指标："99.5% 保真度"、"首次突破..." |
| `metrics.qubit_count` | 量子比特数量，规模化指标 |
| `metrics.fidelity` | 保真度，> 99% 是高质量信号 |
| `metrics.coherence_time` | 相干时间，越长越好 |


---

### 技术成熟度（TRL）判断规则

| TRL | 论文信号 | 投资含义 |
|-----|----------|---------|
| 1–3 | "理论提出"、"首次演示"、单量子比特实验 | 天使/种子，10 年周期 |
| 4–5 | "小规模验证"、"10 量子比特以上"、保真度 > 99% | Pre-A / A 轮 |
| 6–7 | "系统集成"、"量子优势实验"、纠错演示 | A / B 轮窗口 |
| 8–9 | "商业部署"、"客户验证"、重复性保证 | 成长期 |

### 人才投资价值评估框架

**学术影响力评分**：
- `h_index > 50`：国际顶级 PI，定义方向
- `h_index 30–50`：高水平 PI，方向内有决定性影响
- `h_index 10–30`：优秀研究者，团队核心
- 近 3 年论文 > 10 篇：活跃输出中

**产业化信号识别**：
- `position` 含"创始人"、"首席科学家（公司）"、"技术顾问"：直接产业参与
- 研究方向偏向"系统集成"、"可扩展性"、"工程化"：有产业化意识

## 附录：常见问题处理

| 问题 | 处理方式 |
|------|---------|
| `/gold/domains` 的 `paper_count` 全为 0 | 已知 Bug，用 `/papers?domain_ids=X&include_stats=true` 的 `total` 字段替代 |
| 搜索返回结果为空 | 尝试更宽泛的关键词，或去掉 `data_source=seed_data` 参数 |
| API 响应超时 | 单次请求 `page_size` 不超过 100，分页获取 |
| 人才数据覆盖不全 | 组合使用机构查询 + 方向查询，互补盲区 |
