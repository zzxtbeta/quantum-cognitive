
 Tier 0｜系统能“跑起来”的最小闭环（必须一起做）
没有这一层，后面所有智能都是假的
US01｜统一信号流聚合与展示
As 投资经理，我希望在一个“信号流”里看到量子赛道的最新信号，并按类别聚合展示，便于我每天快速扫一遍。
包含类别：论文/专利、融资、工商动态、关键人/团队变迁、政策。
- 前端：Feed 列表 + 类别 Tab/Filter + 时间排序
- 后端：Signal Ingest + Normalization + Feed Query
- 验收标准
  634. 默认展示近 7 天信号，支持按时间范围切换（7/30/90 天）
  635. 每条信号必须有：标题、来源、时间、类别、摘要（<=200字）、关联对象数（公司/人/团队/技术路线）
  636. 支持按类别过滤、按优先级排序

US02｜关键信号识别与优先级排序
As 投资经理，我希望系统默认识别哪些信号“更关键”，并优先展示，减少我做第一轮筛选的时间。
- 前端：Priority 标签（P0/P1/P2 或 高/中/低）+ “关键原因”展开
- 后端：Signal Scoring Service（规则+模型可混合）
- 验收标准
  936. 每条信号有优先级字段 priority
  937. “关键原因”至少返回 1~3 条可读解释（例如：融资额异常、关键团队变动、政策强相关、与已关注标的强关联等）
  938. 用户可手动上调/下调优先级，系统记录反馈

US03｜信号详情页（结构化 + 可追溯）
As 投资经理，我希望点击任一信号可以看到结构化详情，并能追溯来源（原文/原链接/截图），方便我快速判断可信度与下一步。
- 前端：Detail 页面（摘要+要点+原文入口+结构化字段）
- 后端：Signal Detail API + Source Store
- 验收标准
  1241. 详情包含：结构化要点（bullet）、原始来源、提取时间、置信度/来源类型
  1242. 支持“打开原文”
  1243. 支持“标记为噪音/有价值/待验证”

🟡 Tier 1｜认知放大器（你明确点名的核心）
US04｜Chat：统一认知交互中枢（你特别说明的部分）
⚠️ 这是重点：US04 不是一个功能，是一个“交互范式”
As 投资经理，我希望在信号详情页能看到关联的人/团队/公司/技术路线，并能点进去继续探索，形成“认知路径”。
你对 US04 的原始要求（我忠实翻译成工程语言）：
- 它是 Chat 形态
- 接入：数据湖 + 结构化信号 + 实体 + LLM
- 用途不是聊天，而是：
  - 继续探索
  - 关联查找
  - 形成判断支持
US04 的系统定位（非常重要，给研发看的）
US04 = 认知操作系统（Cognitive OS）
它承担三件事：
1087. 把 US01–03 的“静态信息”变成“可对话的认知对象”
1088. 承接用户自然语言的探索意图
1089. 统一未来所有“高级智能”的入口（分析 / 建议 / 发现）
V1 Chat 明确只做 4 件事（非常重要，防止失控）：
- 基于当前信号 / 实体上下文回答问题
- 调用数据湖（RAG）补充信息
- 显式展示引用来源（信号 / 实体）
- 输出的是分析与线索，不是结论
❌ V1 不做 Agent loop
 ❌ V1 不做自动行动
 ❌ V1 不做复杂多轮规划

US05｜记录与关注（形成你的“认知过程”）
As 投资经理，我希望对每条信号都能完成四类动作：
1720. 点开了解；2) 记录为知识更新/进一步探索；3) 关联探索周边；4) 关注人/团队/公司为潜在标的。
- 前端：统一动作栏（Save / Explore / Follow / Ignore）
- 后端：User Action Log + Follow List
- 验收标准
  1876. 每条信号都支持 Save / Follow / Ignore 操作
  1877. Follow 可作用于实体（人/团队/公司）
  1878. 所有动作都写入“认知轨迹”（时间戳+上下文）
工程化解释：
- 信号 / Chat / 实体 → 都能写入同一个笔记系统
- 笔记不是附件，是认知痕迹的一部分
V1 只要求做到
- 从信号详情 / Chat 一键写笔记
- 笔记可反向关联信号 / 实体
- 不做复杂协作（先个人）

🔵 Tier 2｜“这开始像一个真正的投研系统了”
US11｜候选标的发现（Discovery，而不是推荐）
As 投资经理，我希望系统能综合多方面信息，主动形成候选标的列表，并说明“为什么是它”，最终决策仍由我做。
- 前端：Candidate List（排序+原因卡片+一键关注）
- 后端：Candidate Builder（Graph features + scoring）
- 验收标准
  3623. 候选标的至少包含：公司名称、所在地、关联信号数、推荐原因（1~3条）
  3624. 支持按地区（杭州/北京/上海/安徽/深圳等）过滤
  3625. 支持“一键加入关注”，并记录触发信号
