## 字段定义

### 1. name（姓名）
- **类型**：字符串 (string)
- **必填**：是
- **说明**：研究人员姓名，可能包含中英文
- **示例**：
  - `"潘建伟"`
  - `"Katsumi Tanigaki（谷垣勝己）"`
  - `"王浩华"`

### 2. url（个人主页链接）
- **类型**：字符串 (string)
- **必填**：是
- **说明**：研究人员在机构官网的个人主页URL
- **示例**：
  - `"https://www.baqis.ac.cn/people/detail/?cid=615"`
  - `"https://cqi.tsinghua.edu.cn/rydw/jsxl/yaoqizhi.htm"`
  - `"https://person.zju.edu.cn/0010051"`
- **注意**：部分记录可能为空字符串

### 3. title（职称/职位）
- **类型**：字符串 (string)
- **必填**：是
- **说明**：研究人员的职称或职位
- **示例**：
  - `"首席科学家"`
  - `"教授/博士生导师"`
  - `"副研究员"`
  - `"百人计划研究员"`
  - `"院长, 教授"`
  - `"固定岗科研人员"`

### 4. email（电子邮箱）
- **类型**：字符串 (string)
- **必填**：是（但可能为空）
- **说明**：研究人员的工作邮箱
- **示例**：
  - `"pan@ustc.edu.cn"`
  - `"hhwang@zju.edu.cn"`
  - `"katsumitanigaki@baqis.ac.cn"`
- **注意**：部分记录可能为空字符串（如清华大学数据）

### 5. institution（所属机构）
- **类型**：字符串 (string)
- **必填**：是
- **说明**：研究人员所属的研究机构或大学
- **示例**：
  - `"北京量子信息科学研究院"`
  - `"清华大学"`
  - `"中国科学技术大学"`
  - `"浙江大学"`
  - `"粤港澳大湾区量子科学中心"`

### 6. department（所属部门）
- **类型**：字符串 (string)
- **必填**：是（但可能为空）
- **说明**：研究人员所属的院系或研究部门
- **示例**：
  - `"量子物态研究部"`
  - `"交叉信息研究院"`
  - `"量子物理与量子信息研究部"`
  - `"物理学院"`
- **注意**：部分记录可能为空字符串

### 7. research_direction（研究方向）
- **类型**：字符串 (string)
- **必填**：是（但可能为空）
- **说明**：研究人员的主要研究方向或领域
- **示例**：
  - `"算法，密码学，量子计算，人工智能"`
  - `"离子与超导量子计算、量子网络、量子模拟、量子人工智能"`
  - `"超导量子计算和量子模拟"`
  - `"量子单晶材料生长及物态研究"`
  - `"腔QED与量子多体物理"`
- **注意**：部分记录可能为空字符串

### 8. biography（个人简介）
- **类型**：字符串 (string)
- **必填**：是（但可能为空）
- **说明**：研究人员的详细个人简介，包括教育背景、工作经历、研究成果、获奖情况等
- **特点**：
  - 长文本字段，可能包含多段内容
  - 使用 `\n` 作为换行符
  - 内容结构因机构而异
- **示例结构**：
  ```
  【教育背景】
  1972年，哈佛大学，物理学博士学位
  
  【工作经历】
  麻省理工学院数学系，助理教授 (1975年9月至1976年8月)
  
  【研究成果】
  发表论文包括Nature、Science等...
  ```

## 数据质量说明

### 完整性
- **100%完整字段**：所有8个字段在所有记录中都存在（非NULL）
- **可能为空的字段**：
  - `email`：部分机构（如清华大学）的记录中为空字符串
  - `department`：部分记录为空字符串
  - `research_direction`：部分记录为空字符串
  - `biography`：部分记录为空字符串

## 使用建议

### 数据清洗
1. **URL去重**：浙江大学数据中URL重复，需要进一步处理
2. **空值处理**：根据业务需求决定如何处理空字符串
3. **文本规范化**：
   - 统一换行符格式
   - 去除多余空格
   - 标准化职称名称

### 数据整合
建议整合时注意：
- 不同机构的字段完整度不同
- `title` 字段命名规则不统一
- `biography` 格式差异较大

### 推荐用途
- **人才库建设**：构建量子科技领域人才数据库
- **专家检索**：按研究方向、机构、职称检索专家
- **关系网络分析**：分析机构间人才流动和合作关系
- **知识图谱**：构建人物-机构-研究方向知识图谱
